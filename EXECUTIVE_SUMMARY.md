## 🎯 核心的な結論（30秒で読む）

### ✅ テクノロジー・通信セクターは「本物」

```
トータルリターン: +1.64% (唯一のプラス)
日次勝率: 50.0% (22日中11日がプラス)
累積損益: +131万円

→ 「特定日の大勝」ではなく、日々一貫して利益を上げている
→ 過半数の営業日でプラスリターン = 安定性が高い
```

### ❌ 商社セクターは「避けるべき」

```
トータルリターン: -4.52% (最悪)
日次勝率: 31.8% (22日中7日のみプラス)
累積損益: -226万円

→ 68.2%の営業日で損失
→ この戦略には不向き
```

## 🎓 学んだこと

### 技術的な学び

1. **タイムゾーン変換が重要**
   - JST→UTC変換を忘れると0件の取引しか検出できない
   - Refinitiv APIはUTCで動作

2. **PostgreSQLキャッシングの効果**
   - API呼び出し削減: 99%以上
   - 分析再実行が10秒以内に完了
   - 359,000行以上のデータをキャッシュ

3. **Decimal型の扱い**
   - PostgreSQLのNUMERIC型はPythonのDecimal型
   - Pandasとの互換性のためfloat変換が必須

### ドメイン知識の学び

1. **セクター特性**
   - テクノロジー株: 高ボラティリティ、ニュースドリブン
   - 商社: レンジ相場が多い、モメンタムが弱い
   - 自動車: トレンド追従型、ブレイクアウト向きではない

2. **日次勝率の重要性**
   - トータルリターンだけでは一貫性が分からない
   - 日次勝率50%が戦略適合性の目安

3. **個別銘柄バックテストの重要性**
   - プール資金方式: 48取引しか検出できず
   - 個別銘柄方式: 995取引検出（20倍以上）
